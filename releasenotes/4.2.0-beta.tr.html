CotEditor 4.2'de öne çıkanlar

Satır sonlarına hak ettiği değeri verir
CotEditor artık birden çok satır sonu içeren belgeleri olması gerektiği gibi işleyebiliyor. Bir belgedeki satır sonları tutarsızsa CotEditor bunları size bildirir.
İstediğiniz kısımları yazdırın
Yazdırma penceresi, seçilen kısmı yazdırabileceğiniz bir seçenek kazandı. Bununla yalnızca belgenin seçtiğiniz kısmını yazdırabilirsiniz.
Birden çok makinede çalışın
CotEditor artık başka bir makinede iCloud belgeleri ile çalışmayı kolaylaştırmak için Handoff'u destekliyor. Kullanıcı ayarlarını (örn. sözdizim tanımları), başka bir makine ile paylaşmak için Universal Control ile sürükleyip bırakın.

CotEditor 4.2.0-rc

release: 2022-05-03

New Features

  • Support Handoff.

Improvements

  • Update HTML syntax style to display hr elements as separators in the Outline.
  • Restore the file encoding the user explicitly set in the last session.
  • Improve stability.
  • trivial: Enable the secure state restoration introduced in macOS 12.
  • dev: Update Yams from 5.0.0 to 5.0.1.
  • beta: Add the panel notifying about the line ending migration.
  • beta: Select the character in the editor also when an item already selected in the Warnings pane is clicked.
  • beta trivial: Rename “Unicode Next Line” to “Next Line”.

Fixes

  • beta: Fix an issue that the Line Endings submenu in the Format menu did not display minor line ending types even with the Option key.
  • beta: Fix an issue the item fields in the Outline inspector did not resize properly.
  • beta: Fix a possible issue that the initial massage in the incompatible character list did not reflect the actual result.

CotEditor 4.2.0-beta

yayım: 2022-04-27

Bu sürüm, kullanıcılar tarafından bazı belgelerde el ile ayar yapılmasını gerektirebilir. Ayrıntılar için CotEditor 4.2'deki önemli değişiklikler sayfasına bakın.

Yeni Özellikler

  • Birden çok satır sonu içeren belgeleri işleyebilme.
  • Belgeyi açarken tutarsız satır sonlarını bildir.
  • Tutarsız satır sonlarını Denetçi'nin Uyarılar bölümünde listele.
  • Satır sonu seçeneklerine, az kullanılan satır sonları (NEL (New Line), LS (Line Separator) ve PS (Paragraph Separator)) eklendi. Bu seçenekler yalnızca Option düğmesine basıldığında veya belgenin satır sonu bunlardan biriyse görüntülenir.
  • Panoda bulunan metni olduğu gibi yapıştıran ve satır sonlarını belge ayarına göre değiştiren Olduğu Gibi Yapıştır (Command-Option-V) seçeneği eklendi.
  • Yalnızca belgedeki seçili metni yazdırmak için bir seçenek eklendi.
  • Unicode kod noktası girişine geçmiş eklendi.
  • Tercihlerde, tema adı veya çoklu değiştirmeler gibi ayarları dışa aktarmak için ayar adını sürükleyip bırakın.
  • CotEditor'lar arasında ayarları aktarmak için Universal Control ile bir ayar adını sürükleyip başka bir CotEditor'a bırakabilirsiniz.

İyileştirmeler

  • Belge simgeleri güncellendi.
  • Belgelerdeki satır sonları daha kesin bir biçimde algılanır.
  • Birden çok satırlı kırpılmış metni eklediğiniz yerde girintileme düzeni korunur.
  • Yalnızca büyük belgelerde bitişik olmayan metin kipini etkinleştirerek kaydırma davranışını iyileştirir.
  • Sözdizim ayrıştırma iyileştirildi.
  • Uyumsuz Karakterler panelinin adı, tutarsız satır sonlarının da listelenebilmesi için Uyarılar olarak değiştirildi.
  • Yazım yönü sağdan sola iken dikey kaydırıcıyı sol tarafa alır.
  • Yazım yönü sağdan sola iken satır numaralarını sağ tarafa alır.
  • Yazdırırken satır numaralarının dikey konumu iyileştirildi.
  • Son oturum kurtarılırken, belge kodlaması ile uyumlu olmayan karakterleri de kurtar.
  • Uyumsuz karakter listesinde boşlukları boş bırakmak yerine koduyla gösterir.
  • Font ayarı denetimlerine artı/eksi eklendi.
  • Sözdizim tanım dosyaları için .yaml yerine .yml uzantıları kullanılır.
  • Ana hat menü şablonundaki $LN'yi oluşun satır numarası ile değiştirmek için olan özellik artık kullanılmayacak.
  • CoffeeScript ve Tcl özgün belge simgeleri kaldırıldı.
  • Metin daha bir Mac tarzı ifade için gözden geçirildi.
  • Yardım içeriği güncellendi.
  • Önemsiz: Betik menüsü için uzantısında büyük harf olan betikler kabul edilir.
  • Önemsiz: Basit Düzenli İfade sözdizim başvurusunda \n, \R ile değiştirildi.
  • Önemsiz: Anura teması güncellendi.
